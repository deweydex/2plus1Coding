## üöÄ Quick Start Guide

### Before the Workshop

**1 Week Before:**
- [ ] Ensure all students have Claude Code installed
- [ ] Test that Claude Code works on your computers
- [ ] Ensure Python is installed (version 3.7+)
- [ ] Print or share student handouts

**1 Day Before:**
- [ ] Review facilitator guide
- [ ] Decide if you'll assign projects or let students choose
- [ ] Prepare any announcements or slides
- [ ] Test running the sample solutions

**Day Of:**
- [ ] Arrive early to test computers
- [ ] Have student handouts ready (physical or digital)
- [ ] Write workshop schedule on board
- [ ] Prepare for pairs: advanced + beginner

### Workshop Day Schedule

**0:00 - 0:10 (10 min)** - Introduction
- Welcome students
- Explain workshop format
- Form pairs (advanced + beginner)
- Let pairs choose their project (or assign)
- Distribute handouts

**0:10 - 0:25 (15 min)** - Planning Phase
- Pairs discuss and plan their project
- Sketch out game flow on paper
- List core features
- Facilitator circulates to guide

**0:25 - 1:00 (35 min)** - Build MVP
- Students use Claude Code to build
- Start with simplest version
- Test frequently
- Facilitator helps debug

**1:00 - 1:20 (20 min)** - Enhancements
- Add features from handouts
- Polish and improve
- Continue testing

**1:20 - 1:30 (10 min)** - Demo & Reflect
- Pairs swap and play each other's games
- Quick discussion: what worked, what was hard
- Optional: volunteers demo to class

## üéØ Learning Objectives

By the end of this workshop, students will be able to:

1. **Use Claude Code effectively**
   - Craft clear, incremental prompts
   - Ask for explanations
   - Debug with AI assistance

2. **Plan before coding**
   - Break projects into steps
   - Identify core features
   - Design game flow

3. **Test incrementally**
   - Run code after each feature
   - Verify functionality
   - Debug issues as they arise

4. **Read and understand code**
   - Explain what code does
   - Identify functions and their purposes
   - Trace through program flow

5. **Collaborate on coding**
   - Share knowledge between skill levels
   - Explain concepts to partners
   - Make decisions together

## üí° Tips for Success

### For Facilitators

**Do:**
- ‚úÖ Emphasize planning before coding
- ‚úÖ Encourage testing after each change
- ‚úÖ Let students struggle a bit (productive struggle)
- ‚úÖ Celebrate working code, even if simple
- ‚úÖ Help pairs communicate effectively

**Don't:**
- ‚ùå Rush to give answers
- ‚ùå Let advanced students do all the work
- ‚ùå Allow students to skip testing
- ‚ùå Accept "I don't know" without trying
- ‚ùå Let pairs build without understanding

### Common Issues and Solutions

**Issue:** "Advanced student is doing everything"
‚Üí Remind them: "Your job is to teach, not to code"
‚Üí Have beginner type all prompts

**Issue:** "We're stuck and lost"
‚Üí Ask: "What should happen? What's happening instead?"
‚Üí Suggest starting simpler

**Issue:** "We finished too early"
‚Üí Point to enhancement ideas
‚Üí Suggest code review for clarity
‚Üí Have them help another pair

**Issue:** "Our code is broken and we don't know why"
‚Üí Ask them to explain error message
‚Üí Have them describe what they expected
‚Üí Help them form debugging prompt for Claude Code

## üéì Assessment (Optional)

If you want to gauge understanding, ask these questions while circulating:

**For Beginners:**
- "Can you show me where [variable] is stored?"
- "What does this if statement check?"
- "Can you trace through what happens when...?"
- "Why did Claude Code use this approach?"

**For Advanced Students:**
- "How would you explain this to someone new?"
- "What edge cases could break this?"
- "How could this code be improved?"
- "What would you do differently next time?"

## üìû Support During Workshop

If you encounter issues:

1. **Technical Issues**
   - Check Claude Code is running
   - Verify Python installation
   - Restart terminal if needed

2. **Student Confusion**
   - Refer to handouts
   - Use sample solutions as reference (don't show students)
   - Encourage peer help

3. **Time Management**
   - 15 min behind? Skip some enhancements
   - 15 min ahead? Add more enhancements
   - Focus on understanding, not completion
