Getting Started with Claude Code

This page walks you through setting up Claude Code on your machine and completing your first interaction with the tool. Even if you are comfortable with command-line interfaces, take time to work through these steps to understand the specific workflow of Claude Code.

Prerequisites

Before installing Claude Code, you need to ensure your development environment has the necessary foundations. You will need Python installed on your system, as all the projects in this activity use Python. Most modern computers have Python pre-installed, but you may need to install or update it. You can verify whether Python is installed by opening a terminal or command prompt and typing python --version or python3 --version. You should see a version number of 3.8 or higher.

If Python is not installed or needs updating, visit the official Python website at python.org/downloads and download the appropriate installer for your operating system. During installation on Windows, make sure to check the box that says "Add Python to PATH" to ensure you can run Python from any command prompt.

You will also need a text editor or IDE for viewing code. While Claude Code can edit files directly, you will often want to view multiple files or review code in a more comfortable environment. Visual Studio Code, Atom, Sublime Text, or even a simple editor like Notepad++ will work. If you do not have a preferred editor, Visual Studio Code is free and works well for beginners.

Installing Claude Code

Claude Code is installed via npm, the Node Package Manager. If you do not have Node.js and npm installed, you will need to install them first. Visit nodejs.org and download the LTS version for your operating system. The installer includes both Node.js and npm.

Once npm is available, open a terminal or command prompt and install Claude Code globally using the following command: 

npm install -g claude-code

The installation process downloads the necessary files and makes the claude-code command available system-wide. This may take a minute or two depending on your internet connection. If you encounter permission errors on macOS or Linux, you may need to use sudo npm install -g claude-code, though this is not recommended practice. Better to fix your npm permissions first.

To verify the installation was successful, run the following command: 

claude-code --version

You should see a version number displayed. If instead you see an error saying the command is not found, the installation did not complete successfully or your PATH is not configured correctly.

First-Time Configuration

The first time you run Claude Code, you will need to authenticate with Anthropic's servers. Claude Code requires an Anthropic API key, which connects your local tool to Claude's AI models running in the cloud. If you are participating in a workshop, your instructor may provide you with an API key or configure this for you. If you are setting this up independently, you will need to create an account at console.anthropic.com and generate an API key.

Once you have your API key, Claude Code will prompt you to enter it the first time you run the tool. The key is stored securely on your local machine and used to authenticate future requests. You will not need to enter it again unless you switch computers or explicitly reset your configuration.

Note on API keys: Treat your API key like a password. Do not share it publicly or commit it to version control systems like Git. If you accidentally expose your key, you can deactivate it in the Anthropic console and generate a new one.

Creating Your Project Directory

Before starting Claude Code, create a dedicated directory for your project. This keeps your files organized and makes it easier to manage your work. Open a terminal and navigate to where you want to store your project, then create and enter a new directory: 

mkdir my-claude-project
cd my-claude-project

You can name your directory whatever makes sense for your project. For example, if you are building the tic-tac-toe game, you might use mkdir tic-tac-toe-game. The important thing is to have a dedicated space where Claude Code can create and modify files without affecting other parts of your system.

Starting Your First Claude Code Session

With your project directory created and your terminal in that directory, start Claude Code by simply running: 

claude-code

After a brief loading period, you will see a prompt where you can begin typing. This is where you communicate with Claude, asking questions, requesting code, or seeking explanations. The interaction is conversational - you type a request, Claude responds, and you can follow up with additional questions or refinements.

Unlike some command-line tools where you must remember specific commands and flags, Claude Code understands natural language. You can ask questions the same way you would ask a human tutor or pair programmer. The tool maintains context throughout your session, so you can refer back to previous parts of the conversation.

Your First Interaction: A Simple Program

To get comfortable with Claude Code before starting your main project, create a simple program. This helps you understand the workflow without the pressure of building something complex. At the Claude Code prompt, type the following request: 

Create a simple Python script called hello.py that asks for my name and then greets me.

Claude will respond by creating the file and showing you the code it wrote. The tool not only generates code but also explains what it is doing. Read through Claude's response carefully. You will see that it likely created a file with something like this: 

name = input("What is your name? ")
print(f"Hello, {name}! Nice to meet you.")

Now test the program by running it. In your terminal (you may need to open a new terminal window or exit Claude Code temporarily), run: 

python hello.py

The program should prompt you for your name and then greet you. This simple interaction demonstrates the complete cycle: requesting code from Claude, receiving and reviewing the code, and testing it to ensure it works as expected.

Understanding the Claude Code Workflow

The workflow you just experienced is the foundation of working with Claude Code. You make a request in natural language, Claude generates or modifies code, you review and test the code, and then you iterate with follow-up requests. This cycle repeats throughout your development session.

An important principle to understand is that Claude Code operates on your actual files. When Claude creates or modifies a file, those changes are real and permanent. This is different from some AI tools that only show you code suggestions that you must manually copy. Claude Code directly interacts with your file system, which makes development faster but also means you should be thoughtful about reviewing changes before accepting them.

The tool maintains context throughout your session. If you create a file and then ask Claude to modify it later, Claude remembers the file's contents and structure. This allows you to have ongoing conversations about your code rather than repeating context in every request. However, this context is session-specific. When you exit Claude Code and start a new session, you begin with a fresh context.

Practicing Follow-Up Interactions

With your hello.py file still open in Claude Code, try a follow-up request to practice iterating on existing code. Ask Claude something like: 

Can you modify hello.py to also ask for my age and tell me what year I was born?

Claude will update the file with the new functionality. Look at the modified code and think about how it works. If anything is unclear, ask Claude to explain it. For example, you might type: 

Can you explain how the age calculation works in this code?

This back-and-forth interaction is where real learning happens. Do not simply accept code without understanding it. Ask questions, request explanations, and make sure each piece makes sense before moving on.

Key Commands and Interactions

While Claude Code understands natural language, certain phrases and patterns are particularly effective. Here are some common types of requests you will use frequently:

Creating files: "Create a new Python file called game.py" or "Make a file named utils.py with helper functions."

Reading files: "Show me what's in game.py" or "What does the calculate function do in utils.py?"

Modifying code: "Add error handling to the input function" or "Refactor the board display to be more readable."

Running code: "Run game.py" or "Test this code with input of 5."

Debugging: "I'm getting an error that says IndexError on line 12, can you help?" or "This function isn't returning what I expected when I pass it an empty list."

Explaining code: "Explain what this function does line by line" or "Why did you use a dictionary here instead of a list?"

The more specific your requests, the better Claude's responses will be. Rather than "make it better," try "add input validation to ensure the user enters a number between 1 and 9."

Exiting and Resuming

When you want to exit Claude Code, you can type exit or press Ctrl+C (or Cmd+C on Mac). Your files remain in your project directory, so you can return to them anytime. To resume working on your project, simply navigate back to the project directory in your terminal and run claude-code again.

Remember that each Claude Code session starts fresh in terms of conversation context, but your files persist. When you restart, you can ask Claude to review or modify existing files by referring to them by name.

Troubleshooting Common Issues

Claude Code command not found

If your terminal says it cannot find the claude-code command, the installation may not have completed successfully or your PATH environment variable is not configured correctly. Try closing and reopening your terminal first, as PATH changes sometimes require a fresh terminal session. If that does not work, reinstall Claude Code using the installation command above.

Authentication errors

If Claude Code cannot authenticate, verify that your API key is correct and that you have an active internet connection. Claude Code requires network access to communicate with Anthropic's servers. You can reset your configuration and re-enter your API key by deleting the configuration file (location varies by operating system) and restarting Claude Code.

Python version conflicts

Some systems have multiple versions of Python installed. If running python does not work or uses Python 2, try python3 instead. All projects in this activity require Python 3.8 or higher.

Permission errors when creating files

Make sure you are working in a directory where you have write permissions. Your home directory or a dedicated projects folder are good choices. Avoid trying to create files in system directories or protected locations.

Next Steps

With Claude Code installed and working, you are ready to begin your main project. Before diving in, take time to read the Prompting Guide, which explains strategies for effective communication with AI coding assistants. Understanding these principles will help you work more efficiently and learn more effectively.

When you are ready, choose a project from the home page, download the starter code, and begin the planning phase. Remember that the goal is learning through building. Take your time, ask questions, and focus on understanding each step rather than rushing to complete the project.
